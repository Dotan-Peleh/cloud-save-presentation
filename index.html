Why we need this
The Problem

Progress Lost on Reinstall
Players reinstall or switch devices and start from zero. Hours of progress vanish. No recovery path.
🔌
No Cross-Device Sync
Play on one device, nothing carries over. Phone to tablet is a fresh start. Players feel trapped on one device.
📉
Trust & Retention Suffer
Players who lose progress churn. Fear of losing progress reduces engagement and spending confidence.
Core principles

Invisible

🏆
Progress Wins
Higher chapter > higher credits > latest save > cloud wins tie. One method, all cases. Same logic everywhere.

2 Higher credits balance if progress is identical
3 Latest state by sequence_number

Compatibility
Version Management
FLOOR
cloud_save_min_version
Server-side floor — any version >= this is automatically supported. New builds never accidentally excluded. Only bumped on breaking schema changes.
PROMPT
Dismissible Update Prompt
Cloud needs newer version → player sees popup. Can dismiss and keep playing locally. Sync pauses. Re-shown every login until player updates.
RULE
Newer cloud state NEVER loaded on older device
Eliminates unknown items, broken rendering, and crashes. The older client never encounters content it can't handle.
What happens when...
Key Scenarios
📲
Reinstall / New Device
Tutorial skipped entirely. Cloud save restored. "Welcome back! Progress restored" toast shown.
📡
Goes Offline Mid-Session
Toast warns player. Saves locally. Progress-based comparison on reconnect. Higher chapter wins.
🔄
Version Mismatch
Dismissible update popup. Dismiss = play locally, sync pauses. Auto-resumes after updating.
📱📱
Two Devices Simultaneous
Heartbeat ping-pong. Both devices push each other to re-login until one closes. Winner holds session.
🔓
Platform Sign-Out
Gameplay continues unchanged. Sync stops silently. Binding preserved. Auto-reconnects on sign-in.
💾
Save Corrupted
Local corrupted → cloud loaded. Cloud corrupted → local kept. Both → state_history backup. Nothing lost.
Protection
Three Safety Layers
1
Prevention — Min-Floor Model
New versions automatically supported. Silent disconnect only for old versions below the floor. A new build can never be accidentally excluded from cloud save.
2
Detection — Server Alert on Excluded Version
If a newer version is excluded (likely config mistake), server fires alert to Slack/PagerDuty. Team can immediately fix. Player sees update prompt, not silent disconnect.
3
Data Protection — Progress Always Wins
State with higher chapter/credits always wins regardless of timing, upload order, or sequence numbers. Even after days of disconnection, the better state prevails. Losing state always archived.

Architecture
active_states
One row per user — the canonical current state
KEY: distinct_id
state_history
Append-only log of ALL state changes. 30 saves + 7 daily snapshots.
KEY: distinct_id • sequence_number
platform_bindings
Maps platform_id → distinct_id for account linking
KEY: platform_id
session_locks
Active session heartbeats. One session per user.
KEY: distinct_id
device_id_history
All distinct_ids used on each device. For restoring previous ID.
KEY: device_id • distinct_id
device_activity
Tracks last_server_activity per device. Server-side timestamps only.
KEY: device_id
